**Key Technical Concepts**
-   **Outbox Pattern:** Ensures reliable event dispatching by first writing events to a persistent database collection before attempting to send them to a message broker. This prevents data loss if the broker is down.
-   **Celery:** Used as the distributed task queue system to manage background jobs.
-   **Redis:** Serves as the message broker for Celery and the result backend.
-   **Idempotent Consumers:** Consumer tasks are designed to be safely executed multiple times without causing duplicate side effects, using a  collection as a lock.
-   **Supervisor:** Used to manage and automatically restart the Redis, Celery Worker, and Celery Beat processes, ensuring the system is resilient.
